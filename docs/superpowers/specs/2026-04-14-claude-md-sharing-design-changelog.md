### Pass 1 ‚Äî 2026-04-14

1. **Categorization fix: `ls‚Üíeza` aliases and `rmux_helper` demoted from `global.md` to `orbstack-dev.md`.** The source bullets say "on this machine"; putting them in `global.md` would make Claude avoid `ls -t` on a fresh Mac where `ls -t` is correct. Added an explicit "not in global" subsection enumerating what the split keeps out.
2. **`classify_machine` signature refactored to take pre-evaluated booleans** (`mac_ver_nonempty`, `home_developer_exists`) so the test suite never has to mock `pathlib`/`platform`. The previous signature mixed a raw string with a boolean leaking one branch's detection specifics.
3. **`should_install` made a pure computation via a `.enabled` marker file.** Original spec had a circular "opt-in is detected by either symlink exists OR user just approved" clause that `diagnose.py` (pure snapshot producer) cannot compute. Now: marker file presence ‚Üí pure, skill owns the creation of the marker after approval.
4. **Action-kind taxonomy filled in.** JSON example listed only `create_symlink` but Step 3.5 prose referenced `replace_stale_symlink`. Added a full table of four kinds (`create_symlink`, `replace_stale_symlink`, `remove_obsolete_symlink`, `report_user_file`) with emission rules and per-kind skill behavior.
5. **`ln -sfn` ‚Üí `ln -s` for fresh creation.** `-sfn` silently overwrites anything it finds; a race-condition real file would be clobbered. Use plain `ln -s` for `create_symlink` (fails loudly on collision) and reserve `-sfn` for `replace_stale_symlink` where the preceding check already confirmed the entry is a symlink.
6. **`check_post_up_to_date` resolves via `git rev-parse --show-toplevel`, not `os.getcwd()`.** Running `/up-to-date` from a subdirectory of a repo would otherwise miss the hook.
7. **Security subsection for `post-up-to-date.md` prompt-injection.** Executing markdown as an LLM prompt on every sync is an auto-firing execution vector. Added first-sight trust prompt + hash allowlist + re-prompt on change, explicit "never auto-approve".
8. **Unconditional `@-import` for `dev-machine.md` resolved the internal contradiction.** Spec said "safe to include unconditionally" in one paragraph and "only include on machines where detection set up the symlink" in the next. Picked unconditional ‚Äî keeps `~/.claude/CLAUDE.md` byte-identical across machines.
9. **Chop-conventions checkout path risk documented.** Spec JSON hardcoded `/home/developer/...`. Added risk entry: `CHOP_CONVENTIONS_ROOT` env var with `~/gits/chop-conventions` fallback, emit `errors[]` if path is elsewhere rather than silently symlinking to a missing target.
10. **Pre-commit formatter warning added to directory layout.** Prettier will reflow these files on commit; authors should not fight the formatter.
11. **Dropped `claude-md/README.md`.** Would be a third place to describe the split (after spec + skill prose), guaranteed to rot.
12. **Success criteria split into first-run and second-run cases**, plus added a "silent on second run" criterion ‚Äî catches the "enabled marker is idempotent" requirement empirically.

### Pass 2 ‚Äî 2026-04-14

1. **Step 0 test protocol specified concretely.** Pass 1 left "verify `@-import` symlink resolution" as a gate with no protocol. Added a dedicated "Step 0: `@`-import symlink resolution test" subsection under Risks with a 6-step procedure: UUID marker, real target outside `~/.claude/`, symlink creation, `@`-import append, fresh-session probe question, pass/fail criteria, cleanup script, and fail-mode fallback. A UUID marker prevents cached-context false positives. Testing plan now points at this section instead of hand-waving.
2. **`.enabled` marker cross-machine semantics made explicit.** New subsection under "Resolution of `should_install`" nails down: per-machine (because `~/.claude/` is never sync'd), `~/.claude/claude-md/` is a real directory (not a symlink), and marker-exists-but-symlinks-gone is a valid repair state that `diagnose.py` treats as drift without re-prompting opt-in. Closes the ambiguity around what happens if Igor deletes symlinks by hand or enables on only some machines.
3. **`mkdir -p ~/.claude/claude-md` added to the enable flow.** Previous spec assumed the directory existed before writing `.enabled`; on a fresh machine it does not. Now explicit in both the one-time-setup flow (Step 3) and in Step 3.5 (idempotent safety belt).
4. **Error contract for unresolvable chop root.** Pass 1 flagged the `errors[]` emission but didn't say how the skill reacts. Added explicit contract: (a) the rest of `/up-to-date` still runs ‚Äî the subsystems are independent; (b) Step 3.5 is skipped entirely when `shared_claude_md.*` error present; (c) single-line end-of-run notice with the fix instruction; (d) dangling-symlink degraded state is a warning not a fatal. Defined the error entry schema `{subsystem, code, message, probed}` with a concrete code name `chop_root_unresolved`. Step 3.5 wording updated to skip on error.
5. **`hooks-trusted.json` schema, creation, and lifecycle fully specified.** Pass 1 introduced the file with just `{repo_toplevel: sha256}`. Pass 2 adds: version-tagged schema with `entries` map and `{sha256, approved_at, hook_path}` per entry; lazy creation; atomic write via `.tmp` + `os.replace`; corrupt-file handling that re-prompts rather than silently overwriting; no rotation (intentional ‚Äî trust is cheap); explicit per-machine rationale; explicit rejection of the per-repo `.git/info/` alternative. Hash is over raw bytes, no normalization.
6. **Hooks-trusted unit tests added to the testing plan.** Missing file / matching hash / different hash / corrupt JSON / atomic-write exercise ‚Äî covers all branches of the new lifecycle.
7. **Chop-root-unresolved unit test added.** `check_shared_claude_md` with an unresolvable chop root must emit the error entry AND omit the `shared_claude_md` block. Written as a test so the "skip Step 3.5 on error" contract is enforced at the JSON boundary.
8. **Success criterion #7 added: chop-root-unresolvable must not block repo sync.** Captures the subsystem-independence contract as an observable acceptance check. Criterion #5 also expanded to assert the hash-unchanged / hash-changed re-prompt behavior empirically.

### Pass 3 ‚Äî 2026-04-14

1. **`hooks-trusted.json` corrupt-file contract contradiction resolved.** Pass 2 said both "treats every hook as not previously approved and re-prompts" AND "emits an error into `errors[]` and asks Igor to inspect before retrying". Re-prompt vs inspect-before-retry are mutually exclusive ‚Äî a coder would pick one and ship a divergent behavior. Rewrote to explicitly suppress the re-prompt, skip Step 5 post-hook execution, and emit `post_up_to_date.hooks_trusted_corrupt` into `errors[]`. Added the rationale (silent re-prompt ‚Üí single-entry overwrite loses prior trust without notice).
2. **JSON example action list cleaned of mutually-exclusive entries.** The example under `shared_claude_md.actions` listed six actions: `create_symlink` for all three slots AND `replace_stale_symlink` for `machine` AND `remove_obsolete_symlink` for `dev_machine` AND `report_user_file` for `global`. But `actual` showed all three slots with `exists: false`, which contradicts `report_user_file` (requires real file) and `replace_stale_symlink` (requires existing symlink). Reduced to the coherent fresh-install state (three `create_symlink` entries) and added an explicit "at most one action per slot, `actions` has at most three entries" constraint. The other action kinds are documented solely in the kinds table, which is where they belong.
3. **`~/.claude/claude-md` symlink attack closed.** Pass 2 stated "`~/.claude/claude-md/` is a real directory, not a symlink" but relied only on `mkdir -p` to enforce it. `mkdir -p` silently follows a pre-existing symlink, so a compromised or overzealous dotfiles sync (Mackup, chezmoi, a personal setup script) could pre-install `~/.claude/claude-md -> <attacker-controlled path>` and the enable flow would spray `.enabled`, `hooks-trusted.json`, and all slot symlinks into that location. Added an `is_symlink()` check that MUST run before any write under that directory, on both first enable and every subsequent run, and aborts with a clear error if the check fails.
4. **`check_shared_claude_md` signature vs error-emission contradiction resolved by splitting the resolver out.** Pass 1 specified `check_shared_claude_md(chop_root, home, enabled)` ‚Äî a pure function that takes a resolved root. Pass 2 added a unit test "`check_shared_claude_md` with unresolvable chop root emits errors[]" which contradicts the signature (how can the function emit an error for a root it never sees?). Split into `resolve_chop_root(env, home) -> Optional[Path]` and `check_shared_claude_md(chop_root, home, enabled, machine_info) -> (block, errors)`. The orchestrator calls resolve first; on `None` it emits the error and omits the `shared_claude_md` key entirely. Updated the test list to target `resolve_chop_root` for the unresolved case and added an orchestrator-level assertion that the block is absent, not empty. Renumbered the `diagnose.py` steps 1‚Äì6 and fixed the downstream reference under the chop-root risk section.
5. **Symlinked `.claude/post-up-to-date.md` rejected outright.** If the hook file is a symlink ‚Äî tracked or not ‚Äî it can point at an arbitrary file on the local filesystem whose contents change independently of the repo's commit history, breaking the hash-allowlist's only security guarantee (hash changes ‚áí re-prompt ‚áí Igor sees the new content before execution). A file at `~/.bashrc` or another repo's hook could be substituted at any time with no `git pull` involvement. `diagnose.py` now emits `post_up_to_date.hook_is_symlink` and reports `post_up_to_date_path: null`. Also specified "read once, hash once, execute once" to close the TOCTOU window between hash check and LLM feed ‚Äî the skill must read the hook into memory a single time and feed those same bytes to both the hasher and the LLM, never re-opening.
6. **Dangling-symlink degraded warning grounded in a real signal.** Pass 2's error contract said "if `.enabled` exists and chop root is unresolvable, that is a degraded-state warning (existing symlinks are dangling)" ‚Äî but pass 2 also said the `shared_claude_md` block is omitted in that case, so the skill has no JSON path to learn whether symlinks are dangling. Specified that the skill stat-checks `.enabled` directly (one call, no dependency on the omitted block) and appends a second line to the end-of-run notice explaining the dangling state. No changes to the no-delete / no-reset invariants.

### Pass 4 ‚Äî 2026-04-14

1. **Stale `hooks-trusted.json` corrupt-file test description rewritten.** The testing-plan entry still read "corrupt/unparseable JSON ‚Üí re-prompt all hooks AND error surfaced" ‚Äî the exact pass 2 contract that pass 3 explicitly overturned in change #1. A coder following the test plan would have implemented re-prompt-on-corrupt, re-introducing the overwrite-loses-prior-trust bug pass 3 closed. Rewrote the entry to match pass 3's contract: Step 5 skipped, `post_up_to_date.hooks_trusted_corrupt` emitted, no re-prompt, corrupt file not overwritten. Also tightened the atomic-write clause to name the happy path it exercises.
2. **TOCTOU test added to enforce "read once, hash once, execute once".** Pass 3 mandated in-memory byte reuse between the hasher and the LLM feed to close the TOCTOU window between approval and execution, but added no test. Without a test the contract is prose-only ‚Äî a future refactor that re-opens the hook file for LLM feeding would silently regress without tripping CI. Added a dedicated testing-plan entry: mock `open`/`read_bytes` on the hook path, assert call count == 1, assert the bytes reaching the hasher and the LLM feed are identical, plus a second case that mutates the on-disk file between the mocked read and execution and asserts the in-memory bytes (not the mutated file) reach the LLM.
